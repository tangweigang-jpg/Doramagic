## 审查结果：需要修改的条目

以下条目存在分类偏差，需要调整 `kind` 分类：

### 需要从 `operational_lesson` 改为 `domain_rule` 的条目

这些条目描述的是工具/框架的客观规律，不遵守会导致直接错误或系统崩溃：

**[13]** `operational_lesson` → **`domain_rule`**
- **upgrade_reason**: 描述 TuShare Pro API 的客观要求——必须配置 TUSHARE_API_TOKEN 才能发起 API 调用。这是框架的硬性契约，不遵守会导致认证失败。
- **severity 校准**: `fatal`（正确）— 无 token 则整个数据层完全不可用。

**[14]** `operational_lesson` → **`domain_rule`**
- **upgrade_reason**: 描述速率限制器的客观行为——超过限制会被限流/封禁。0.9 系数是安全边界，不是"经验教训"而是框架硬性约束。
- **severity 校准**: `high`（当前 medium → 建议提升）— 超出限流会导致数据获取完全失败，影响交易决策。

**[74]** `operational_lesson` → **`domain_rule`**
- **upgrade_reason**: 描述技术指标计算的数学要求——少于 20 个数据点则统计结果无效（无效样本量）。这是技术分析框架的客观限制。
- **severity 校准**: `high`（当前 medium → 建议提升）— 用无效数据做信号判断会导致错误交易。

**[75]** `operational_lesson` → **`domain_rule`**
- **upgrade_reason**: 相关性矩阵计算需要至少 2 个标的。这是统计计算的数学前提，不是经验教训。
- **severity 校准**: `high`（当前 medium → 建议提升）— 持仓 < 2 时强行计算相关性无意义。

**[76]** `operational_lesson` → **`domain_rule`**
- **upgrade_reason**: 风险指标计算需要足够的收益率样本量（20 天）。这是统计有效性的客观要求。
- **severity 校准**: `high`（当前 medium → 建议提升）— 数据不足时风险指标误导性极强。

**[105]** `operational_lesson` → **`domain_rule`**
- **upgrade_reason**: 描述数据 API 接口契约——必须去除 SH/SZ 前缀才能正确查询。代码化接口要求，不是踩坑总结。
- **severity 校准**: `high`（当前 high → 正确）。
