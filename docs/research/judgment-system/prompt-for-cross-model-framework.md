# 提示词：AI 顾问知识结构框架构建

## 提示词

```
我正在设计一种叫"种子晶体"的知识格式。背景如下：

## 背景

今天的大语言模型（LLM）能写代码，但在特定领域中，它的产出缺少生产级质量。比如让LLM"做一个股票投资分析工具"，它会生成能运行的代码，但会犯很多领域专家不会犯的错误：用float做财务计算、用免费API做实时交易决策、回测时忽略幸存者偏差等。

我们的假设是：LLM和生产级产出之间的差距，本质上是"约束知识"的缺失——不是不知道怎么做，而是不知道什么不能做、为什么不能做。但这只是我们当前的假设，如果你认为差距的本质不是这个，或者不仅仅是这个，请直接指出。

## 种子晶体的概念

种子晶体是一个结构化的知识包，大小随需求复杂度而变，供用户的AI工具消费，让AI在生成代码时自动遵守领域约束。实验证明：同一个LLM，有种子晶体时在8项生产特性中命中7项，无种子晶体时命中0项。

种子晶体还是一个可传播的消费品——用户A用了觉得好，会直接转发给朋友B。B不需要理解晶体的内容（就像不需要读懂App的源码），只需要把它放进自己的AI工具里就能获得同样的质量提升。这意味着框架设计需要考虑：知识如何组织才能让同一个晶体在不同用户、不同AI工具中都有效。

## AI顾问的知识来源

AI顾问的知识不是凭空产生的。它的知识来自对大量相关开源项目的深度代码分析（不是只读README，而是全量代码分析）。比如，为了构建"股票投资工具"的种子晶体，AI顾问会分析 freqtrade、zipline、vnpy 等多个开源量化项目的实际代码、Issue、PR、社区讨论。

关键方法是"跨项目合成"：当多个独立项目在同一问题上采用相同做法时（共识信号），这可能是可靠的行业约束；当它们做法不同时（分歧信号），这是需要深入分析的争议点。

## 我的产品

我在构建一个"AI顾问"——它是一个AI形态的行业专家，类似麦肯锡顾问：用户描述需求后，AI顾问根据对大量开源项目的深度理解，为用户编译出一个种子晶体。用户拿着这个种子晶体去自己的任何AI工具中使用。顾问交付判断（种子晶体），用户的本地AI负责执行（生成代码）。

## 你的任务

我需要你为这个AI顾问设计一套知识结构框架。具体来说：

以"帮我做一个股票投资分析工具"为测试场景，回答：

1. **AI顾问需要哪些类型的知识？**（比如约束、资源、模式、还是其他？）
2. **这些知识有哪些层级？**（比如通用→行业→系统→交叉，还是你有更好的分层方式？）
3. **每个类型×层级的交叉点，具体包含什么知识？** 用股票投资工具的真实例子填充。
4. **知识之间的关系是什么？** 特别是：资源选择如何产生约束？不同层级的知识如何碰撞出新的交叉知识？
5. **哪些知识是LLM自己就有的（不需要提供），哪些是必须外部注入的？** 划出边界。
6. **我的前提假设有没有问题？** "差距=约束缺失"这个判断是否完整？种子晶体作为知识载体是否遗漏了什么？

## 要求

- 不要泛泛而谈，用股票投资工具的具体例子来填充你的框架
- 要体现知识之间的"生成关系"（A的选择导致B成为约束）
- 框架要能推广到其他领域（不只是股票），但用股票做具体示例
- 如果你认为我的分类（约束/资源/模式）有遗漏或错误，请直接指出并提出你的分类
- 请以markdown格式输出你的框架
```
