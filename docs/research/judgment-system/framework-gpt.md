# GPT 的 AI 顾问知识结构框架

> 来源：GPT (Codex)，2026-04-02

---

## 核心判断

"差距=约束缺失"抓住主轴但不是全貌。更准确：

```
生产级差距 = 约束缺失 + 判据缺失 + 证据缺失 + 取舍策略缺失 + 验证闭环缺失 + 运行时事实缺失
```

种子晶体不应只是"知识文档"，而应是：
> 种子晶体 = 目标契约 + 世界模型 + 资源画像 + 决策规则 + 实现蓝图 + 验证闭环 + 证据出处

## 知识类型（7类）

| 类型 | 作用 | 股票工具例子 |
|---|---|---|
| T1 目标契约 | 定义什么叫成功、什么不做 | 这是分析工具，不是自动交易系统 |
| T2 世界模型 | 领域对象、因果关系、边界条件 | 交易日历、复权、成分股变迁、财报发布日期 |
| T3 资源画像 | 可用数据/库/服务及其能力边界 | Yahoo Finance只适合EOD研究 |
| T4 决策规则 | 该做/不该做/何时做 | 不用float、回测必须建模滑点 |
| T5 实现蓝图 | 推荐架构与可复用实现模式 | ingest→normalize→feature→simulate→evaluate→report |
| T6 验证闭环 | 如何自动发现违规和退化 | 未来函数检测、幸存者偏差检查 |
| T7 证据出处 | 规则来源、置信度、适用范围 | "费用建模"是多框架强共识；事件驱动vs向量化是分歧 |

## 知识层级（5层）

| 层级 | 含义 | 股票工具例子 |
|---|---|---|
| L0 通用工程层 | 所有生产软件通用 | 可复现、可测试、可审计 |
| L1 领域现实层 | 资本市场客观规律 | 交易日历、企业行为、流动性 |
| L2 任务产品层 | 这次要做的具体产品 | 股票投资分析工具侧重研究与评估 |
| L3 系统实现层 | 当前资源下的技术实现 | Python+DuckDB+Parquet+因子回测 |
| L4 运行治理层 | 合规、可运维、可传播 | 数据许可、免责声明、版本化 |

## 知识生成关系

### 资源选择→约束衍生
- 选免费EOD数据 → 不能做盘中策略 → 报告必须显示数据滞后
- 选pandas float记账 → 必须用Decimal → 增加"资金守恒"测试
- 选"当前股票列表" → 回测不能宣称无幸存者偏差

### 层级碰撞→交叉知识
- 领域层×任务层：财报发布日期滞后 → 因子回测必须point-in-time join
- 领域层×系统层：拆股改变价格 → 必须拆成raw和adjusted两表
- 任务层×运行层：产品可传播 → 工具必须自动暴露数据新鲜度警告

## LLM已有 vs 必须注入

LLM已有但不可信到生产级：适用条件不稳定、优先级不稳定、容易漏关键项。

必须注入：当前任务产品边界、资源能力与禁区、高价值隐性约束、共识规则与分歧点、质量判据、验证器与反例、数据许可。

额外类别"运行时事实"：不能靠静态晶体注入（如API是否可用、数据是否新鲜），需要"能力契约"（假定有哪些资源、需要哪些验证器、缺失时如何降级）。

## 推荐最小晶体结构（YAML原型）

包含：intent、world_model、resources、rules(hard_constraints+soft_policies)、blueprints、validators、evidence、degradation

## 最终定位

AI顾问卖的不是"帮用户总结知识"，而是：
> 把分散、隐性、带条件的行业判断，编译成可传播、可执行、可验证的质量约束包。
