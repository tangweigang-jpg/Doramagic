# "AI 复制项目经验能不能做" — 人话版

> 同目录下 `2026-04-22-knowledge-replication-feasibility.md` 是学术完整版（含论文引用、SWE-bench 数据等）。本文是给 CEO 看的白话翻译。

---

## 一句话答案

**能做，但天花板不高。你现在正走在天花板上——该换地方挖矿。**

---

## 为什么天花板不高

用"厨艺"打比方。项目经验分三层：

**第一层：菜谱**
几斤几两、几分钟、炒几下。写下来，谁都能照着做。
→ **AI 能学会，但这种东西 ChatGPT 早学过了**。你再写一份，它不感谢你。

**第二层：为什么这么做**
"东北菜必须用东北大酱不用韩国大酱。"老师傅说得出，但要讲很久。
→ **AI 能学会，但需要你给它超大量实例**。你写一条约束让 AI 看一眼，它记不住。

**第三层：看火候**
老师傅瞥一眼就知道火大了。他自己也说不清怎么看的。
→ **AI 2026 年学不会。不是工程问题，是物理问题。**

Doramagic 挖的主要是第二层。**但这一层已经被 LLM 训练时吃掉了 80-90%**，你只能挤出剩下的 10-20%。

---

## Doramagic "4pp" 这个数字什么意思

你做了个实验：
- 不给 AI 晶体，它答业务问题得 **45 分**
- 给它晶体，得 **49 分**
- 涨了 **4 分**

是不是觉得晶体做得不够好？**不是**。

对比：2023 年有一个著名实验叫 SWE-bench。给 AI **整个项目的完整代码 + 文档**，它的表现只涨了 **2 分**。

你的 4 分比这个还多一点。但**数量级一样**。

**结论：你晶体做得再精致，物理上能涨的就这么多。** 继续做 v7、v8、v9 晶体，涨不动。

---

## 为什么是物理上限

你在 GitHub 上挖金矿。**但金子早被 LLM 挖走了。**

LLM 训练的时候把 GitHub 扫了一遍。你选"AI 项目做抄作业大师"——这恰好是 LLM 最熟的地盘。你晶体里装的 90% 内容，它已经知道。

---

## 另一个致命问题：你给的是建议，不是命令

v1 为什么死？你生成了一个 Python 脚本，让用户的 AI 调。**AI 嫌麻烦，自己瞎写**。

v2 换了 YAML + SKILL.md。但 AI 读这份说明书时，仍然可以**选择不听**。它觉得自己能搞定就自己搞。

**Anthropic 自己承认了这个问题**：他们公开说 "SKILL.md 本质是 prompt，依赖模型自愿遵从"。GitHub 上有个 Issue #157 叫"Executable Agent Skills"正在提案——就是想修这个问题。**但还没修好**。

所以 v1 死在这里，v2 也死在这里，只是换了个漂亮外套。

---

## 那什么能做？三个条件

三个至少中两个才能 work：

**1. 有自动裁判**
做对做错能自动判。比如：代码跑测试通过/不通过，合规文件格式对/不对，审计发现有/无违规。**没有裁判，AI 乱写你也不知道。**

**2. 用户 AI 硬管着它**
不是"建议 AI 这么做"，而是**"不这么做 AI 就跑不起来"**。现在 Claude Code / Cursor 都没这个能力。得等 Executable Skills 提案落地，或者你自己做一个这样的 host。

**3. 窄到一针见血**
只做一件事，做到底。Devin 只做安全漏洞修复，效率比人快 20 倍。**试图承载"整个项目经验"的东西，最后都会稀释到 4pp 形状。**

**Doramagic 现在打分：0/3。**

## 两条活路

### 路 A：换地方挖矿

别在开源 AI 项目上卷。找 LLM 没看过的领域：
- 企业内部流程（跨部门审批、风控流程）
- 医药合规（新药申报、GMP 质检）
- 金融监管（反洗钱、信披、税务筹划）
- 工业 SOP（生产质检、设备点检）

共同特点：**私密 + 不规整 + 有明确对错**。

LLM 训练数据里没这些。你做出来才有真正的 delta，不是 4pp 而是可能 20pp+。

### 路 B：换形态

别做"给别人 AI 的说明书"，直接做"帮用户干活的 agent"。

参考 Devin：
- **窄场景**（只做 A 股回测、只做 MACD 策略）
- **自动裁判**（代码跑回测、结果能对比 benchmark）
- **直接输出结果**（不是教 AI 怎么做，是帮用户做完）

晶体降级为**你内部开发 agent 的原材料**，不再作为对外交付物。用户拿到的是一个跑起来就出结果的 agent，不是一份 SKILL.md。

---

## 不建议的路

**继续做晶体 → Skills 市场分发**：
- 4pp 是信号不是噪音
- SkillsMP 800K skills 里 60-70% 被放弃——你做了第 800001 个没人用
- 行业共识方向（Executable Skills）还没落地，你押早了

---

## 我的综合判断

**停止晶体的"外部分发"定位。** 至少暂停 v7/v8/v9 迭代。

**选一条活路**：

- 如果你更擅长**选领域 + 做生意**：走路 A（换私密领域，晶体继续做但卖给企业）
- 如果你更擅长**做技术产品**：走路 B（做 Devin 式窄 agent，晶体变内部资产）

**两条路不互斥**，但第一步只能选一条先走。
