# 多方评审 Prompt：知识 + 骨架混合模式

> 复制以下内容，分别发给 GPT / Grok / Gemini 获取独立评审意见。

---

## 背景

我们做的产品叫 Doramagic，核心产出是"晶体"——一个给 AI 注入领域知识的单文件配方（Markdown 格式，4000+ 行）。用户把这个文件丢给任意 AI（如 MiniMax、Claude、GPT），AI 读取后获得领域专业知识，能帮用户完成专业任务。

当前架构是**纯知识注入**：所有蓝图、约束、API 示例都作为知识文本写入晶体。AI 读取理解后，自主生成代码。

## 问题

在 A 股量化回测领域的实测中发现：**AI 能理解安全规则（如 T+1 交割限制、涨跌停过滤），但不保证在生成的代码中实现它们。**

实测数据（3 次回测，宿主 AI 为 MiniMax-M2.7）：
- T+1 交割限制：1/3 实现（v2.0 后首次自主实现，之前缺失）
- 涨跌停过滤：0/3 实现（三次全部缺失）
- 产出数据质量校验（assert 语句）：0/3 实现

我们尝试了以下方案但均未完全解决：
- 在晶体中写 `[FATAL] 必须实现涨跌停过滤` → AI 理解了但不执行
- 提供安全检查函数模板让 AI copy-paste → AI 理解了但自己重写（有时漏了涨跌停）
- 用 YAML 控制块声明状态机和验证规则 → AI 把它当知识内化，不机械执行

## 提案

在晶体中新增一个"回测骨架"（Scaffold）段：一个 Python 代码模板，安全检查（T+1、涨跌停、印花税、整手限制）已经焊死在里面，AI 只需要在 `REPLACE_WITH` 占位符处填入策略逻辑。

```
当前：AI 从知识中理解规则 → 自主生成代码（可能漏掉安全检查）
提案：AI 拿到骨架 → 填入策略逻辑（安全检查已内置，不可删除）
```

## 请评审以下 5 个问题

1. **你认为"AI 理解了知识但不保证在代码中实现"是一个普遍问题，还是我们的晶体写得不够好？** 换句话说，是否存在某种 prompt 写法能让 AI 100% 实现安全检查，还是说这是 LLM 的结构性局限？

2. **骨架模式（Scaffold）是一个好的解决方案吗？** 它的核心假设是：与其让 AI "理解后自主实现"，不如给它一个框架让它"在限定范围内填空"。你认同这个方向吗？有没有更好的方案？

3. **骨架模式最大的风险是什么？** 比如：AI 会不会直接忽略骨架重新生成代码？骨架会不会限制 AI 的灵活性？

4. **如果你来解决这个问题，你会怎么设计？** 不限于我提出的方案，可以提出全新的思路。

5. **一句话结论**：这个提案应该采用、拒绝、还是有条件采用？

请用中文回答，每个问题 3-5 句话。
